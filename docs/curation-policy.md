# Curation Policy

## Acceptance standard

A resource should meet most of these conditions:

- Clearly relevant to AI builders
- Publicly accessible
- Documented enough to evaluate
- Still active or historically important
- Useful beyond a single novelty use case

## Ranking preference

When multiple similar resources exist, prefer the one with:

- Better documentation
- Better examples
- Better maintenance
- Better licensing clarity
- Better fit for real developer workflows

## Rejection cases

- Broken links
- Spam or affiliate-first pages
- Repackaged duplicates
- Thin prompt lists with no explanation
- Tools that cannot be evaluated from public information

## Editing standard

- Use short summaries
- State the concrete use case
- Keep marketing language out
- Keep metadata normalized
