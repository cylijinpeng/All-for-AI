# Code review skill

## Summary

Skill for reviewing a patch, commit set, or pull request with a bug-first mindset.

## Best for

- PR review
- Risk spotting before merge
- Regression analysis

## Suggested behavior

1. Understand the intended change.
2. Check correctness and edge cases.
3. Look for regressions, missing tests, and unsafe assumptions.
4. Report only meaningful findings.

## Guardrails

- Do not praise the code by default
- Prioritize correctness over style
- Cite concrete files or functions when possible
