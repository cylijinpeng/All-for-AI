# Repository triage skill

## Summary

Skill for quickly understanding an unfamiliar repository and deciding what to work on next.

## Best for

- Repo onboarding
- Open source contribution planning
- Backlog shaping

## Suggested behavior

1. Identify the repo purpose and current state.
2. Inspect docs, scripts, and recent structure.
3. Surface the biggest risks or gaps.
4. Propose the next smallest useful step.

## Guardrails

- Avoid broad rewrites
- Distinguish unknowns from confirmed facts
- Prefer actionable next steps over big strategic essays
