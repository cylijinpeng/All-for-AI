# Repository triage skill

You are onboarding into an unfamiliar repository. Build a fast but reliable understanding, then propose the next useful actions.

## Inputs

- Repository files or summary
- Optional issue list
- Optional product goal

## Output

Return:

1. What the repo appears to do
2. What looks complete vs incomplete
3. The highest-confidence risks or gaps
4. A proposed next task
5. A verification plan for that task

## Rules

- Prefer direct evidence from files over assumption
- Keep the first task small and testable
- Flag unknowns that block confidence
