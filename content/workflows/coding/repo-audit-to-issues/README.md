# Repository audit to issues workflow

## Goal

Inspect a repository and turn observations into a small, prioritized issue backlog.

## Inputs

- Repository files or link
- Product goal
- Contributor constraints

## Steps

1. Understand the repo purpose.
2. Inspect docs, setup, code shape, and tests.
3. Identify missing capabilities or risks.
4. Convert the highest-confidence gaps into issues.
5. Order the issues by leverage and difficulty.

## Deliverables

- Technical audit summary
- Prioritized issue list
- Suggested first issue
