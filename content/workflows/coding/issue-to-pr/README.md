# Issue to PR Workflow

## Goal

Workflow for turning a scoped issue into a validated implementation and pull request summary.

## Steps

1. Clarify the issue and acceptance criteria
2. Inspect the codebase and find the smallest safe change surface
3. Plan the implementation
4. Make the change
5. Run validation
6. Write a concise PR summary

## Deliverables

- A concrete output artifact
- A short validation note
- A clear next step if more work is needed
