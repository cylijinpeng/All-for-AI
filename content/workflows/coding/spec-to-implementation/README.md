# Spec to Implementation Workflow

## Goal

Workflow for converting a written spec into an implementation plan, code changes, and verification.

## Steps

1. Read the spec and identify the concrete deliverable
2. Map affected files and systems
3. Plan staged implementation
4. Implement the smallest useful slice
5. Verify against the spec

## Deliverables

- A concrete output artifact
- A short validation note
- A clear next step if more work is needed
