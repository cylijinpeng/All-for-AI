# Prompts

Prompts that can be copied into GitHub Models, IDE agents, or custom apps.

## Original starter prompts

- [`coding/debug-bug`](coding/debug-bug/README.md): turn a bug report into a debugging plan
- [`coding/pr-review`](coding/pr-review/README.md): review a patch with a code-review mindset
- [`coding/repo-audit`](coding/repo-audit/README.md): inspect a repository and produce an audit
- [`coding/architecture-review`](coding/architecture-review/README.md): review a design and surface the highest-value risks
- [`coding/refactor-plan`](coding/refactor-plan/README.md): turn cleanup intent into a staged refactor plan
- [`coding/implementation-plan`](coding/implementation-plan/README.md): convert a feature request into an implementation plan
- [`coding/test-strategy`](coding/test-strategy/README.md): design a lean test plan
- [`coding/migration-plan`](coding/migration-plan/README.md): plan a safe migration across boundaries
- [`coding/incident-postmortem`](coding/incident-postmortem/README.md): turn incident notes into a postmortem
- [`research/paper-summary`](research/paper-summary/README.md): turn a paper into an implementation-oriented summary
- [`research/deep-research-brief`](research/deep-research-brief/README.md): shape a broad topic into a research brief
- [`research/competitor-analysis`](research/competitor-analysis/README.md): compare products or approaches meaningfully
- [`writing/project-brief`](writing/project-brief/README.md): shape a rough idea into a project brief
- [`writing/feature-spec`](writing/feature-spec/README.md): turn a feature idea into a spec
- [`writing/user-story-pack`](writing/user-story-pack/README.md): turn a feature into user stories and acceptance criteria
- [`analysis/eval-plan`](analysis/eval-plan/README.md): design an evaluation plan for prompts, agents, or apps
- [`automation/mcp-server-design`](automation/mcp-server-design/README.md): scope an MCP server around a tool or data source
